## Runtime contents of the shaded SMT JAR

The Maven Shade plugin creates a Kafka Connect plugin JAR.

Kafka Connect dependencies are marked as `provided`, so Kafka Connect API libraries are expected to be supplied by the Kafka Connect runtime image and are not bundled by the current build.

The Protocol Buffers Java runtime is shaded into the plugin JAR.

## Container image note

This repository builds the SMT JAR. It does not, by itself, contain or redistribute the complete Kafka Connect runtime image.

If you build and distribute a Kafka Connect container image that includes this SMT plus additional runtime components, the image may also include third-party components such as:

```text
Strimzi Kafka Connect base image
Apache Kafka / Kafka Connect runtime
Stream Reactor MQTT connector
Confluent Amazon S3 Sink Connector
Linux distribution packages from the base image
```

Those components have their own licenses and notices. Add their license files and notices to the image or accompanying documentation according to their redistribution requirements.

Suggested image-level layout:

```text
/licenses/
  envsensor-kafka-smt-LICENSE.txt
  protobuf-LICENSE.txt
  apache-2.0-LICENSE.txt
  confluent-community-license-1.0.txt
  strimzi-LICENSE.txt
  kafka-LICENSE.txt
  kafka-NOTICE.txt
  stream-reactor-LICENSE.txt
/THIRD_PARTY_NOTICES.md
```

Adjust this list to match the actual image contents.

## Practical release checklist

Before publishing a source release:

- [ ] Keep the repository `LICENSE` file as the MIT License for project-owned source code.
- [ ] Keep this `THIRD_PARTY_NOTICES.md` file.
- [ ] Keep Kafka dependencies as `provided` if Kafka Connect runtime libraries should not be bundled.
- [ ] Verify the shaded JAR contents:

```bash
jar tf target/*-all.jar | sort > target/jar-contents.txt
```

- [ ] Check that the shaded JAR contains `com/google/protobuf/...`.
- [ ] Check that the shaded JAR does not contain Kafka Connect runtime classes unless dependency scopes were intentionally changed.
- [ ] Include the protobuf license notice with binary/JAR releases.
- [ ] Do not include real Kafka topics, S3 credentials, registry tokens, GitHub tokens, or production payloads in source or binary releases.

Before publishing a container image:

- [ ] Collect license and notice files for the base image and every connector plugin copied into `/opt/kafka/plugins`.
- [ ] Include notices for Strimzi, Kafka, Stream Reactor, Confluent Amazon S3 Sink Connector, Protocol Buffers, and any OS packages required by the image.
- [ ] Do not bake deployment secrets into the image.
- [ ] Provide credentials through Kubernetes Secrets, environment variables, private Helm values, or a secret manager.
